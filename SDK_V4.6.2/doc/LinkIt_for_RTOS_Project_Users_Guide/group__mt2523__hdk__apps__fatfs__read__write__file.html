fatfs_read_write_file
mt2523_hdk » apps
Overview
  • Application description
    • This application demonstrates how to use FatFS to access a file stored on SD/eMMC.
    • The application does not require FreeRTOS.
  • Results
    • Log will show whether the file is read and written successfully.
Hardware and software environment
  • Supported platform
    • LinkIt 2523 HDK.
  • HDK switches and pin configuration
    • Connect J1001.2 to J1001.3;
    • Connect J1002.2 to J1002.3;
    • Connect J1003.2 to J1003.3;
    • Connect J1004.2 to J1004.3;
    • Connect J1005.2 to J1005.3;
    • Connect J1006.2 to J1006.3;
  • Note:
    • It's recommended to apply correct configuration for the switches and pins to ensure successful operation of the SD/eMMC and FatFs based applications. The EPT configuration of GPIO30/GPIO31/GPIO32/GPIO33/GPIO34/GPIO35 for this application is to enable SD/eMMC and it is different from common EPT configuration.
  • Environment configuration
    • The output logs are communicated through a micro USB cable to the PC from MK20 USB connector on the HDK.
    • Install the mbed serial driver according to the instructions at https://developer.mbed.org/handbook/Windows-serial-configuration. For more information, please refer to section "Installing the LinkIt 2523 HDK drivers on Microsoft Windows" on the "LinkIt 2523 HDK User Guide" in [sdk_root]/doc folder.
    • Use a type-A to micro-B USB cable to connect type-A USB of the PC and MK20 micro-B USB connector on the LinkIt 2523 HDK. For more information about the connector cable, please refer to https://en.wikipedia.org/wiki/USB#Mini_and_micro_connectors.
    • Launch a terminal emulator program, such as Tera terminal on your PC for data logging through UART. For the installation details, please refer to section "Installing Tera terminal on Microsoft Windows" on the "LinkIt for RTOS Get Started Guide" in [sdk_root]/doc folder.
    • COM port settings. baudrate: 115200, data: 8 bits, stop bit: 1, parity: none and flow control: off.
Directory contents
  • Source and header files
    • src/main.c: Main program.
    • src/system_mt2523.c: MT2523x clock configuration file.
    • src/sys_init.c: MT2523x hardware configuration file.
    • src/ept_eint_var.c: MT2523x EINT configuration file.
    • src/ept_gpio_var.c: MT2523x PinMux configuration file. The file with a prefix of "ept_" is generated by Easy Pinmux Tool (EPT), with the *.dws file under ept_config folder of a project.
    • inc/task_def.h: Defines the task stack size, queue length, project name, and priority for the application to create tasks.
    • inc/hal_feature_config.h: MT2523x feature configuration file.
    • inc/memory_map.h: MT2523x memory layout symbol file.
    • GCC/startup_mt2523.s: MT2523x startup file for GCC.
    • GCC/syscalls.c: MT2523x syscalls for GCC.
    • MDK-ARM/startup_mt2523.s: MT2523x startup file for Keil IDE.
    • EWARM/startup_mt2523.s: MT2523x startup file for IAR embedded workbench.
  • Project configuration files using GCC
    • GCC/feature.mk: Feature configuration.
    • GCC/Makefile.: Makefile.
    • GCC/flash.ld: Linker script.
  • Project configuration files using Keil IDE
    • MDK-ARM/fatfs_read_write_file.uvprojx: uVision5 project file. Contains the project structure in XML format.
    • MDK-ARM/fatfs_read_write_file.uvoptx: uVision5 project options. Contains the settings for the debugger, trace configuration, breakpoints, currently open files, etc.
    • MDK-ARM/flash.sct: Linker script.
  • Project configuration files using IAR embedded workbench
    • EWARM/fatfs_read_write_file.ewd: IAR project options. Contains the settings for the debugger.
    • EWARM/fatfs_read_write_file.ewp: IAR project file. Contains the project structure in XML format.
    • EWARM/fatfs_read_write_file.eww: IAR workspace file. Contains project information.
    • EWARM/flash.icf: Linker script.
Run the application
  • Build the example project with the command "./build.sh mt2523_hdk fatfs_read_write_file" from the SDK root folder and download the binary file to LinkIt 2523 HDK.
  • Connect the HDK to the PC with a type-A to micro-B USB cable, and open the port with the name "mbed Serial Port". Connect all the switches described in "HDK switches and pin configuration" section and insert an SD card.
  • Run the application. The transaction result is displayed in the log and "The file is written and read successfully!" indicates a successful access to a file.
