freertos_create_thread
mt7697_hdk » templates
Overview
  • Example description
    • This example describes how to setup FreeRTOS multitasking environment. The project creates four tasks. Each task sleeps for a certain amount of time measured in OS ticks and prints the logs when it awakes.
  • Process / procedure of the example project
    • Declare and initialize four tasks and schedule at different intervals.
  • Results
    • Log will show when a task wakes up.
Hardware and software environment
  • Supported HDK
    • MediaTek LinkIt 7697 HDK.
  • HDK switches and pin configuration
    • N/A.
  • Environment configuration
    • The output logs are communicated through a type-A to micro-B USB cable to the PC from MK20 USB connector on the HDK.
    • Install the mbed serial driver according to the instructions at https://developer.mbed.org/handbook/Windows-serial-configuration. For more information, please refer to section "Installing the LinkIt 7697 HDK drivers on Microsoft Windows" on the "LinkIt 7697 HDK User Guide" in [sdk_root]/doc folder.
    • Use a type-A to micro-B USB cable to connect type-A USB of the PC and MK20 micro-B USB connector on the LinkIt 7697 HDK. For more information about the connector cable, please refer to https://en.wikipedia.org/wiki/USB#Mini_and_micro_connectors.
    • Launch a terminal emulator program, such as Tera terminal on your PC for data logging through UART. For the installation details, please refer to section "Installing Tera terminal on Microsoft Windows" on the "LinkIt for RTOS Get Started Guide" in [sdk_root]/doc folder.
    • COM port settings. baudrate: 115200, data: 8 bits, stop bit: 1, parity: none and flow control: off.
Directory contents
  • Source and header files
    • src/main.c: Main program.
    • src/system_mt7687.c: The configuration file of the Cortex-M4 with floating point core registers and system clock
    • src/sys_init.c: This file initializes the hardware environment for the system, including system clock, UART port for logging, Pinmux, cache and other necessary hardware.
    • src/ept_eint_var.c: The EINT configuration file generated by Easy Pinmux Tool (EPT). Please do not edit the file.
    • src/ept_gpio_var.c: The GPIO configuration file generated by the EPT. Please do not edit the file.
    • inc/FreeRTOSConfig.h: FreeRTOS feature configuration file.
    • inc/task_def.h: Define the task stack size, queue length, project name, and priority for the application to create tasks.
    • inc/hal_feature_config.h: MT76x7 feature configuration file.
    • inc/flash_map.h: MT76x7 memory layout symbol file.
    • inc/ept_eint_drv.h: The EINT configuration file generated by the EPT. Please do not edit the file.
    • inc/ept_gpio_drv.h: The GPIO configuration file generated by the EPT. Please do not edit the file.
    • inc/sys_init.h: The header file of sys_init.c.
    • GCC/startup_mt7687.s: MT76x7 startup file for GCC.
    • GCC/syscalls.c: The minimal implementation of the system calls.
    • MDK-ARM/startup_mt7687.s: MT76x7 startup file for Keil IDE.
  • Project configuration files using GCC
    • GCC/feature.mk: Feature configuration.
    • GCC/Makefile.: Makefile.
    • GCC/mt7687_flash.ld: Linker script.
  • Project configuration files using Keil IDE
    • MDK-ARM/freertos_create_thread.uvprojx: uVision5 Project File. Contains the project structure in XML format.
    • MDK-ARM/freertos_create_thread.uvoptx: uVision5 project options. Contains the settings for the debugger, trace configuration, breakpoints, currently opened files, etc.
    • MDK-ARM/flash.sct: Linker script.
Run the demo
  • Build the example project with a command "./build.sh mt7697_hdk freertos_create_thread" from the SDK root folder and download the binary file to LinkIt 7697 HDK.
  • Connect the HDK to the PC with a type-A to micro-B USB cable and specify the port on Tera terminal corresponding to "mbed Serial Port".
  • Run the example. The log will show "hello world x", where x is the task ID with the range from 0 to 3.
