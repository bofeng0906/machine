fatfs_read_write_file
mt7687_hdk » apps
Overview
  • Application description
    • This application demonstrates how to use FatFS to access files stored on SD/eMMC. This project does not require FreeRTOS.
  • Features of the application
    • This application is a reference to create a single file then write and read data to and from it.
  • Results
    • Log will show whether the file is read and written successfully.
Hardware and software environment
  • Supported platform
    • LinkIt 7687 HDK.
  • HDK switches and pin configuration
    GPIOx PINx
    7 J36.3
    24 J36.4
    25 J36.5
    26 J36.6
    - Note
    • Connect J36.3 to SD card DAT3;
    • Connect J36.4 to SD card CMD;
    • Connect J36.5 to SD card DAT0;
    • Connect J36.6 to SD card CLK;
    • Connect J32.2 to SD card VDD;
    • Connect J21.2 to SD card VSS;
    • Connect J21.2 to SD card VSS;
    • It's recommended to apply correct configuration for the switches and pins to ensure successful operation of the SD/eMMC and FatFs based applications. The EPT configuration of GPIO7/GPIO24/GPIO25/GPIO26 for this application is to enable eMMC based on SPI and it is different from common EPT configuration.
  • Environment configuration
    • The output logs are communicated through a micro USB cable to the PC from USB (CON5) connector on the HDK.
    • Install the mbed serial driver according to the instructions at https://developer.mbed.org/handbook/Windows-serial-configuration. For more information, please refer to section "Installing the LinkIt 7687 HDK drivers on Microsoft Windows" on the "LinkIt 7687 HDK User Guide" in [sdk_root]/doc folder.
    • Use a type-A to micro-B USB cable to connect type-A USB of the PC and MK20 micro-B USB connector on the LinkIt 7687 HDK. For more information about the connector cable, please refer to https://en.wikipedia.org/wiki/USB#Mini_and_micro_connectors.
    • Launch a terminal emulator program, such as Tera terminal on your PC for data logging through UART. For the installation details, please refer to section "Installing Tera terminal on Microsoft Windows" on the "LinkIt for RTOS Get Started Guide" in [sdk_root]/doc folder.
    • COM port settings. baudrate: 115200, data: 8 bits, stop bit: 1, parity: none and flow control: off.
Directory contents
  • Source and header files
    • src/main.c: Main program.
    • src/system_mt7687.c: MT76x7 clock configuration file.
    • src/ept_eint_var.c: MT76x7 EINT variable name configuration file.
    • src/ept_gpio_var.c: MT76x7 GPIO variable name configuration file.
    • inc/ept_eint_drv.h: MT76x7 EINT configuration file.
    • inc/ept_gpio_drv.h: MT76x7 GPIO configuration file. The file with a prefix of "ept_" is generated by Easy Pinmux Tool (EPT), with the *.dws file under ept_config folder of a project.
    • inc/task_def.h: Defines the task stack size, queue length, project name, and priority for the application to create tasks.
    • inc/hal_feature_config.h: MT76x7 feature configuration file.
    • inc/flash_map.h: MT76x7 memory layout symbol file.
    • GCC/startup_mt7687.s: MT76x7 startup file for GCC.
    • GCC/syscalls.c: MT76x7 syscalls for GCC.
    • MDK-ARM/startup_mt7687.s: MT76x7 startup file for Keil IDE.
    • EWARM/startup_mt7687.s: MT76x7 startup file for IAR embedded workbench.
  • Project configuration files using GCC
    • GCC/feature.mk: Feature configuration.
    • GCC/Makefile.: Makefile.
    • GCC/mt7687_flash.ld: Linker script.
  • Project configuration files using Keil IDE
    • MDK-ARM/fatfs_read_write_file.uvprojx: uVision5 project file. Contains the project structure in XML format.
    • MDK-ARM/fatfs_read_write_file.uvoptx: uVision5 project options. Contains the settings for the debugger, trace configuration, breakpoints, currently open files, etc.
    • MDK-ARM/flash.sct: Linker script.
  • Project configuration files using IAR embedded workbench
    • EWARM/fatfs_read_write_file.ewd: IAR project options. Contains the settings for the debugger.
    • EWARM/fatfs_read_write_file.ewp: IAR project file. Contains the project structure in XML format.
    • EWARM/fatfs_read_write_file.eww: IAR workspace file. Contains project information.
    • EWARM/flash.icf: Linker script.
Run the application
  • Build the application with the command "./build.sh mt7687_hdk fatfs_read_write_file" from the SDK root folder and download the binary file to the LinkIt 7687 HDK.
  • Connect your board to the PC with a micro USB cable, connect an SD card to your board.
  • Connect all the switches described in "HDK switches and pin configuration" section.
  • Run the application. The transaction result is displayed in the log and "The file is written and read successfully!" indicates a successful access to one file.
