lwip_socket
mt7687_hdk » apps
Overview
  • Application description
    • This application shows the usage of socket APIs.
  • Features of the application
    • This application demonstrates how to create a UDP/TCP client/Server and how to communicate between client and server.
    • This application explains how to:
    • 1. Connect to an access point.
    • 2. Create TCP/UDP server and wait client to connect.
    • 3. Create TCP client to connect to a server and send data to the server. The server echoes the received data back to the client.
    • 4. Create UDP client to connect the server and send data to the server. The server echoes the received data back to the client.
  • Results
    • The output is in a log.
Hardware and software environment
  • Supported platform
    • LinkIt 7687 HDK.
  • Environment configuration
    • The output logs are communicated through a micro USB cable to the PC from USB (CON5) connector on the HDK.
      • Install the mbed serial driver according to the instructions at https://developer.mbed.org/handbook/Windows-serial-configuration. For more information, please refer to section "Installing the LinkIt 7687 HDK drivers on Microsoft Windows" on the "LinkIt 7687 HDK User Guide" in [sdk_root]/doc folder.
      • Use a type-A to micro-B USB cable to connect type-A USB of the PC and MK20 micro-B USB connector on the LinkIt 7687 HDK. For more information about the connector cable, please refer to https://en.wikipedia.org/wiki/USB#Mini_and_micro_connectors.
      • Launch a terminal emulator program, such as Tera terminal on your PC for data logging through UART. For the installation details, please refer to section "Installing Tera terminal on Microsoft Windows" on the "LinkIt for RTOS Get Started Guide" in [sdk_root]/doc folder.
      • COM port settings. baudrate: 115200, data bits: 8, stop bit: 1, parity: none and flow control: off.
      • Configure a Wi-Fi router with the SSID as SQA_TEST_AP, password as 77777777, and authentication mode as WPA_PSK_WPA2_PSK. The Wi-Fi router should have the internet connection.
      • You can change the SSID and password in main.c to match these of your Wi-Fi Access Point.
Directory contents
  • Source and header files.
    • src/main.c: Main program file.
    • src/system_mt7687.c: The configuration file of the Cortex-M4 with floating point core registers and system clock.
    • src/sys_init.c: This file initializes the basic hardware environment for the system, including system clock, UART port for logging, Pinmux, cache and other necessary hardware.
    • src/wifi_lwip_helper.c: Network helper function for Wi-Fi station mode initialization until IP network ready.
    • src/ept_eint_var.c: The EINT configuration file generated by Easy Pinmux Tool (EPT). Please do not edit the file.
    • src/ept_gpio_var.c: The GPIO configuration file generated by the EPT. Please do not edit the file.
    • inc/FreeRTOSConfig.h: FreeRTOS feature configuration file.
    • inc/task_def.h: Define the task stack size, queue length, project name, and priority for the application to create tasks.
    • inc/hal_feature_config.h: MT76x7 feature configuration file.
    • inc/flash_map.h: MT76x7 memory layout symbol file.
    • inc/ept_eint_drv.h: The EINT configuration file generated by the EPT. Please do not edit the file.
    • inc/ept_gpio_drv.h: The GPIO configuration file generated by the EPT. Please do not edit the file.
    • inc/sys_init.h: The header file of sys_init.c.
    • inc/wifi_lwip_helper.h: The header file of wifi_lwip_helper.c.
    • inc/lwipopts.h: The configuration file of tcpip feature.
  • Project configuration files using GCC
    • GCC/feature.mk: Feature configuration.
    • GCC/makefile: Makefile.
    • GCC/mt7687_flash.ld: Linker script.
    • GCC/startup_mt7687.s: MT76x7 startup file for GCC.
    • GCC/syscalls.c: The minimal implementation of the system calls.
  • Project configuration files using Keil
    • MDK-ARM/lwip_socket.uvprojx: uVision5 project file. Contains the project structure in XML format.
    • MDK-ARM/lwip_socket.uvoptx: uVision5 project options. Contains the settings for the debugger, trace configuration, breakpoints, currently open files, etc.
    • MDK-ARM/flash.sct: Linker script.
  • Project configuration files using IAR
    • EWARM/lwip_socket.ewd: IAR project options. Contains the settings for the debugger.
    • EWARM/lwip_socket.ewp: IAR project file. Contains the project structure in XML format.
    • EWARM/lwip_socket.eww: IAR workspace file. Contains project information.
    • EWARM/flash.icf: Linker script.
Run the application
  • Build the application with the command, "./build.sh mt7687_hdk lwip_socket" from the SDK root folder and download the bin file to LinkIt 7687 HDK.
  • Connect the HDK to the PC with a micro USB cable.
  • Run the application. The result is displayed in the log. "example project test success" printed in the log indicates a success.
