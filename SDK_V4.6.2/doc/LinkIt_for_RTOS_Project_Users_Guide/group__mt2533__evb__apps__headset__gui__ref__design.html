Overview

Overview
  • Application description
    • This application demonstrates headset features based on MediaTek MT2533 SOC for RTOS development platform. The architecture diagram is shown below.
    • Application features
      • Sink. Enables to connect with a remote device, handle calls and play music from the remote device.
      • Battery Service (BAS). Provides the battery status of MT2533 to a remote device.
      • Apple Notification Center Service (ANCS). Provides a simple and convenient way to access the notifications generated on the iOS devices.
      • Bluetooth Notification. Receives and displays notifications from Android devices.
      • Digital Clock. Displays a digital clock on idle screen.
      • Main Menu. Processes pen event assignment and provides an entry point for sub-features.
      • Audio Player. Plays local MP3 music.
      • Firmware update Over The Air (FOTA). Uses Bluetooth notification service to receive FOTA packages and trigger an update on the target board.
      • Find Me Profile (FMP). Allows users to find the misplaced device using Bluetooth LE.
      • The application features are independent, if not otherwise mentioned.
Hardware and software environment
  • Supported platform
    • MT2533 evaluation board (EVB).
  • Environment configuration
    • Please contact MediaTek customer support team to find more information regarding the pin configuration, hardware peripheral setup, debugger setup and more.
    • Note that when LCM changes its state from normal mode to idle mode, the screen is slightly flashing. This issue is caused by the driver IC of RM67160, the watch LCM module.
  • Customization options and usage
    • No compile option is available to disable an application feature.
    • To disable a feature, delete the feature initialization function in the main.c.
  • Concurrent feature operation
    • Sink and audio player can operate concurrently. Although, the priority of sink is higher than that of audio player. When Bluetooth EDR connection is established for the sink, the audio track will be stopped and any action on audio player will not be responded.

Run the application
  • How to build the headset_ref_design application
    • GCC version
      • make command "./build.sh mt2533_evb headset_gui_ref_design" under the SDK root folder.
    • Keil IDE
      • Open "headset_gui_ref_design.uvprojx" under the MDK-ARM folder and build the Keil project.
    • IAR embedded workbench IDE
      • Open "headset_gui_ref_design.ewp" under the EWARM folder and build the IAR project.
  • How to download the headset_gui_ref_design application
    • Install and use Flash Tool ("sdk_root/tools/PC_tool_Win.zip") to download the application binary headset_gui_ref_design.bin.
  • How to run each feature
    • Power on the MT2533 EVB to initialize and run the features in the background. The idle screen will be shown with a digital clock on the display.
    • Touch the idle screen to enter the main menu with two icons.
    • Click the Bluetooth icon to enable the Bluetooth, or click the FMP client icon to run the Find Me client feature.
    • Users also can use keypad to trigger sink and audio player actions.
    • BAS, ANCS, Blueooth Notification, FOTA and Find Me server can also be triggered on the smartphone. Find more details in each sub-feature project's readme file.
